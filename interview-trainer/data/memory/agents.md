# Interview Trainer Agent Rules

## Project Goal

This project is a local interview preparation system. The app in `interview-trainer/` builds topic indexes, generates interview papers, records spoken/text answers, grades answers with Codex, writes feedback, and updates memory.

## Directory Permissions

- `八股/` is mutable. It contains interview-focused knowledge and may receive generated or consolidated Markdown documents.
- `笔记/` is read-only. In the public demo repo it should stay empty except for a placeholder file. Personal notes may be added locally and must never be modified by the app.
- `interview-trainer/data/` stores runtime indexes, papers, feedback, memory, tasks, RAG chunks, and question cache.

## Generation Rules

- Interview questions must be natural-language questions generated by Codex from retrieved context or reused from the high-quality question bank.
- Do not create paper questions by directly wrapping Markdown titles, headings, or structured fields in templates.
- If Codex fails and there are not enough cached questions for the selected topic, fail the task instead of returning low-quality template questions.
- Follow-up mode should create 3-5 related questions around one root concept with increasing depth.

## Retrieval Rules

- Do not send the whole knowledge base to Codex for every paper.
- Rebuild the lightweight RAG index from `八股/*.md` and optional local `笔记/*.md`, then retrieve a limited set of chunks by topic, keyword, memory weakness, and randomness.
- `note_readonly` chunks may influence generated questions and grading, but write-back candidates must target `八股/` only.

## Task Queue

- Codex-backed jobs run through the local FIFO task queue.
- Topic/RAG rebuild, paper generation, grading, and 八股 document generation must not run concurrently.
- Persist task status so browser refreshes do not lose queued/running/completed/failed state.

## Memory Rules

- Memory records topic, knowledge point, question, and follow-up chain performance.
- Correct, partial, and wrong answers should update mastery level and review priority.
- Bad or ambiguous generated questions should be marked so they are not reused.

## 八股 Document Generation

- Given technology keywords and a question count, generate a new Markdown interview document only under `八股/`.
- Generated documents should include overview, high-frequency questions, reference answers, follow-up directions, common mistakes, and interview phrasing advice.
- Existing files should not be overwritten by default; use a timestamped filename when needed.
